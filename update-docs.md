# /update-docs

Update all project documentation to reflect recent changes.

## Instructions

1. Review recent git commits and changes made in this session
2. Update `docs/architecture.md` if any structural changes were made (new components, schema changes, new integrations)
3. Update `docs/changelog.md` with a new entry under `[Unreleased]` describing what was added, changed, or fixed
4. Update `docs/project-status.md`:
   - Mark completed tasks as ✅
   - Update "Where We Left Off" with today's date, last completed task, and next task
   - Note any new decisions made
5. Review `CLAUDE.md` — if any new rules, commands, or constraints were established during this session, add them
6. Create a git commit with message: `docs: update project documentation`

Be concise. Don't over-document. Only update what actually changed.
