# /retro

Run a retrospective on the current development session to drive continuous improvement.

## Instructions

1. Review what was built or attempted in this session
2. Reflect on the following:
   - What went well?
   - What was slow, repetitive, or frustrating?
   - Were there any mistakes Claude made more than once?
   - Were there any prompts or workflows that worked especially well?
3. Based on the retro, make improvements:
   - Add new rules or constraints to `CLAUDE.md` to prevent repeated mistakes
   - Update or create slash commands if a workflow could be automated
   - Note any improvements to the planning or setup process
4. Summarize the retro findings in a short bullet list
5. Commit any changes with: `chore: retro improvements`

The goal is for every session to make the next session faster and smoother.
